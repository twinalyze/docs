#### Events Automatically Tracked

These events are tracked by default (no code required):

<table class="event-table">
  <thead>
    <tr>
      <th>Event Name</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Screen Viewed</strong></td>
      <td>Automatically logs activity-level screen views (e.g., `HomeActivity`, `LoginActivity`).</td>
    </tr>
  
    <tr>
      <td><strong>Fragment Viewed</strong></td>
      <td>Logs views for individual fragments within an activity, e.g., `ProductListFragment`.</td>
    </tr>
  
    <tr>
      <td><strong>Tab View</strong></td>
      <td>Tracks user switching between bottom or custom tab components (e.g., Home, Cart, Profile).</td>
    </tr>
  
    <tr>
      <td><strong>Drawer View</strong></td>
      <td>Tracks drawer open/close events, useful for identifying navigation engagement.</td>
    </tr>
  
    <tr>
      <td><strong>Dialog</strong></td>
      <td>Tracks dialog open/close events (e.g., alerts, pop-ups, confirmation dialogs)</td>
    </tr>
  
    <tr>
      <td><strong>App in Foreground</strong></td>
      <td>Fires when the app enters the foreground (e.g., user opens or resumes the app).</td>
    </tr>
  
    <tr>
      <td><strong>App Language</strong></td>
      <td>Tracks the appâ€™s current language on first launch or when changed by the user. Useful for understanding localization trends.</td>
    </tr>
  
    <tr>
      <td><strong>Splash Time</strong></td>
      <td>Captures time spent on the splash screen to help analyze loading performance.</td>
    </tr>
  
  </tbody>
</table>

#### Manual Tracking (Fallback)

If auto-tracking fails on certain devices or flows, use the following manual methods to ensure data is captured.

#### Screen Viewed

Call this method in `onCreate()`.

<CodeGroup>

```javascript Kotlin / Java
SetAnalytics.getInstance()
  .setActivityEvent(
    screenName: String,
    screenClass: Activity
  )
```

</CodeGroup>

<Accordion title="Property Definitions">

<table class="event-table">
  <thead>
    <tr>
      <th>Property Key</th>
      <th>Description</th>
      <th>Example Value</th>
      <th>Required</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`screenName`</td>
      <td>Display name of the screen</td>
      <td>`Main Activity`</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>`screenClass`</td>
      <td>Full class name of the activity screen</td>
      <td>`com.MainActivity`</td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>
 
</Accordion>


Example of Screen Viewed

<CodeGroup>

```javascript Kotlin
SetAnalytics.getInstance()
    .setActivityEvent(
        "MainActivity",    // screenName
        this@MainActivity // screenClass
    )
```

```javascript Java
SetAnalytics.getInstance()
    .setActivityEvent(
        "MainActivity", // screenName
        this           // screenClass
    );
```

</CodeGroup>

#### Fragment Viewed

Call this method in `onCreateView()`.

<CodeGroup>

```javascript Kotlin / Java
SetAnalytics.getInstance()
  .setFragmentEvent(
    screenName: String,
    screenClass: Activity,
    fragmentClass: Fragment,
    fragmentIndex: Int
  )
```

</CodeGroup>

<Accordion title="Property Definitions">

<table class="event-table">
  <thead>
    <tr>
      <th>Property Key</th>
      <th>Description</th>
      <th>Example Value</th>
      <th>Required</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>`screenName`</td>
      <td>Display name of the screen</td>
      <td>`Home`</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>`screenClass`</td>
      <td>Activity class hosting the fragment</td>
      <td>`com.mainActivity`</td>
      <td>Optional</td>
    </tr>
    <tr>
      <td>`fragmentClass`</td>
      <td>Fully qualified class name of fragment</td>
      <td>`com.home`</td>
      <td>Yes</td>
    </tr>
    <tr>
      <td>`fragmentIndex`</td>
      <td>Position index of the fragment within the stack</td>
      <td>`0`</td>
      <td>Yes</td>
    </tr>
  </tbody>
</table>
 
</Accordion>

Example of Fragment Viewed

<CodeGroup>

```javascript Kotlin (Example 1)
activity?.let {
    SetAnalytics.getInstance()
        .setFragmentEvent(
            "fragmentName",    // screenName
            it,                // screenClass
            this@FragmentOne,  // fragmentClass
            0                  // fragmentIndex
        )
}
```

```javascript Kotlin (Example 2)
activity?.let { activity ->
  SetAnalytics.getInstance()
      .setFragmentEvent(
          "fragmentName",     // screenName
          activity,           // screenClass
          this@FragmentOne,   // fragmentClass
          0                   // fragmentIndex
          )
}
```

```javascript Java
SetAnalytics.getInstance()
    .setFragmentEvent(
        "fragmentName",   // screenName
        getActivity(),    // screenClass
        this,             // fragmentClass
        0                 // fragmentIndex
    );
```

</CodeGroup>
